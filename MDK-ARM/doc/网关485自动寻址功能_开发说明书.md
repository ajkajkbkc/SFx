## 1. 概述

### 1.1 功能目标

网关通过 RS-485 总线，自动发现并给所有未配置的从站设备分配站号地址（1~N），无需人工干预。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **不中断现有通信** | 在空闲时段或系统启动时触发寻址，不影响已有的数据采集 |
| **兼容已有从站** | 已配置站号的从站不受影响，旧固件从站忽略寻址帧 |
| **最多支持 255 个从站** | 实际建议 ≤ 32 个（485总线负载限制） |
| **断电保存** | 从站列表保存到 Flash 参数区，重启后恢复 |
| **自动补新** | 支持后续新增从站的再次寻址，不覆盖已有站号 |

### 1.3 工作流程总览

```
用户触发/上电自动触发
        │
        ▼
┌─────────────────────┐
│  广播 0x64 开始寻址  │  ──→ 所有未配置从站进入寻址模式
└─────────┬───────────┘
          │
          ▼ (循环)
┌─────────────────────┐
│  广播 0x65 查询下一个  │  ──→ 未配置从站随机延时回复令牌
└─────────┬───────────┘
          │
     ┌────┴────┐
     │ 收到响应 │  超时
     └────┬────┘
          │            │
          ▼            ▼
┌─────────────────┐  ┌─────────────────────┐
│ 分配地址 + 等待确认 │  │  所有从站已分配完毕   │
└─────────┬───────┘  └─────────┬───────────┘
          │                    │
     ┌────┴────┐              │
     │ 确认成功 │  失败重试     │
     └────┬────┘  最多3次     │
          │                    │
          ▼                    ▼
     ┌──────────────────────────────────────┐
     │  广播 0x68 结束寻址 + 保存从站列表    │
     └──────────────────────────────────────┘
```

## 3. 通信协议定义

### 3.1 帧格式（Modbus-RTU）

```
┌─────────┬─────────┬──────────────┬──────────┐
│ 地址域   │ 功能码  │   数据域      │  CRC16   │
│  1 Byte  │ 1 Byte  │   N Bytes    │  2 Bytes │
└─────────┴─────────┴──────────────┴──────────┘
```

- 广播地址 = `0x00`
- CRC16 = 标准 Modbus CRC16（多项式 `0x8005`）

### 3.2 功能码定义

| 功能码 | 名称 | 方向 | 数据域 | 说明 |
|--------|------|------|--------|------|
| `0x64` | `START_ADDR` | 网关→广播`(0x00)` | 无 | 开始自动寻址 |
| `0x65` | `NEXT_QUERY` | 网关→广播`(0x00)` | 无 | 查询下一个未配置从站 |
| `0x66` | `ASSIGN_ADDR` | 网关→广播`(0x00)` | `[Token(2B)] + [站号(1B)]` | 分配地址 |
| `0x67` | `ADDR_ACK` | 从站→网关`(单播)` | `[站号(1B)]` | 地址确认回复 |
| `0x68` | `END_ADDR` | 网关→广播`(0x00)` | 无 | 结束自动寻址 |

### 3.3 协议交互时序图

```
网关                                           从站A          从站B
 │                                               │              │
 │  ─── 0x64 [广播, 开始寻址] ─────────────────→ │              │
 │                                               │(未配置,进入)  │(未配置,进入)
 │                                               │              │
 │  ─── 0x65 [广播, 查询下一个] ───────────────→ │              │
 │                                               │ T_A=134ms    │ T_B=512ms
 │                                               │ Token=0xA3B2 │ Token=0x7C1E
 │                                               │              │
 │                            (134ms后)          │              │
 │  ?── Token=0xA3B2 ─────────────────────────── │              │
 │                                               │              │(猜到总线忙,取消)
 │                                               │              │
 │  ─── 0x66 [广播, Token=0xA3B2, Addr=1] ────→ │              │
 │                                               │ 匹配! 保存!   │ 不匹配,忽略
 │  ?── 0x67 ACK(Addr=1) ────────────────────── │              │
 │                                               │              │
 │  记录: 从站1 已分配                            │              │
 │                                               │              │
 │  ─── 0x65 [广播, 查询下一个] ───────────────→ │              │
 │                                               │(已配置,忽略)  │ T_B=67ms
 │                                               │              │ Token=0x4F91
 │                            (67ms后)           │              │
 │  ?── Token=0x4F91 ────────────────────────── │              │
 │                                               │              │
 │  ─── 0x66 [广播, Token=0x4F91, Addr=2] ────→ │              │
 │                                               │ 不匹配,忽略   │ 匹配! 保存!
 │  ?── 0x67 ACK(Addr=2) ────────────────────────────────     │
 │                                               │              │
 │  ... 重复直到超时 ...                                       │
 │                                               │              │
 │  ─── 0x68 [广播, 结束寻址] ─────────────────→ │              │
 │                                               │ 退出寻址模式  │ 退出寻址模式
 │                                               │              │
 │  保存从站列表到 Flash                                        │
 │  从站总数: 2                                                 │
```

### 3.4 时序参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 波特率 | 9600 bps | 485 总线常用（根据实际配置） |
| 查询超时 `QUERY_TIMEOUT` | 2000 ms | 发出 0x65 后等待令牌响应的最长时间 |
| ACK 超时 `ACK_TIMEOUT` | 1000 ms | 发出 0x66 后等待 0x67 确认的最长时间 |
| 重试次数 `MAX_RETRY` | 3 次 | 分配地址失败后的最大重试次数 |
| 帧间隔 `FRAME_GAP` | ≥ 50 ms | 连续两帧之间的最小间隔 |
| 结束前空闲等待 `IDLE_WAIT` | 3000 ms | 连续 N 次查询无响应后才判定寻址结束 |

## 4. 网关状态机设计

### 4.1 状态定义

```c
typedef enum {
    GW_ADDR_IDLE,              // 空闲状态
    GW_ADDR_STARTING,          // 准备开始（发送 0x64）
    GW_ADDR_QUERYING,          // 查询中（发送 0x65 + 等待响应）
    GW_ADDR_ASSIGNING,         // 分配中（发送 0x66 + 等待确认）
    GW_ADDR_RETRY_ASSIGN,      // 重试分配
    GW_ADDR_CHECK_COMPLETE,    // 检查是否完成
    GW_ADDR_FINISHING,         // 收尾（发送 0x68 + 保存）
} GwAddrState_t;
```

### 4.2 状态转换图

```
                   用户触发/上电自动启动
                           │
                           ▼
                    ┌───────────┐
                    │   IDLE    │
                    └─────┬─────┘
                          │
                          ▼
                    ┌───────────┐
                    │ STARTING  │ ──→ 发送 0x64 广播
                    └─────┬─────┘    然后切换到 QUERYING
                          │
                          ▼
              ┌───────────────────────┐
         ┌───│      QUERYING          │ ?────┐
         │   │  (发送 0x65 + 等待)    │      │
         │   └───────────┬───────────┘      │
         │               │                  │
         │          ┌────┴────┐             │
         │          │         │             │
         │     收到令牌    超时(2s)        │
         │          │         │             │
         │          ▼         │             │
         │   ┌──────────┐    │             │
         │   │ ASSIGNING│    │             │
         │   │ (分配地址)│    │             │
         │   └────┬─────┘    │             │
         │        │          │             │
         │   ┌────┴────┐     │             │
         │   │         │     │             │
         │  ACK收到  超时    │             │
         │   │         │     │             │
         │   │    ┌────┴────┐              │
         │   │    │RETRY_   │── 重试>3? ──→│
         │   │    │ASSIGN   │  是→跳过     │
         │   │    └─────────┘              │
         │   │                             │
         │   └──→ CHECK_COMPLETE          │
         │              │                  │
         │         ┌────┴────┐             │
         │         │         │             │
         │    还有从站  连续超时           │
         │         │     N次(如3次)        │
         │         │         │             │
         └─────────┘         ▼             │
                        ┌───────────┐      │
                        │ FINISHING │ ─────┘
                        │ (发0x68+  │
                        │  保存)    │
                        └───────────┘
                              │
                              ▼
                         回到 IDLE
```

### 4.3 状态详细说明

| 状态 | 说明 | 进入动作 | 退出条件 |
|------|------|---------|---------|
| `IDLE` | 空闲，不进行寻址 | 初始化 / 寻址结束 | 触发寻址（按键/命令/上电自动） |
| `STARTING` | 发送开始广播 | 发送 `0x64` + 启动 `FRAME_GAP` 定时器 | 帧发送完成 → QUERYING |
| `QUERYING` | 发送查询并等待响应 | 发送 `0x65` + 启动 `QUERY_TIMEOUT` 定时器 | 收到令牌 → ASSIGNING；超时 → CHECK_COMPLETE |
| `ASSIGNING` | 发送分配并等待确认 | 发送 `0x66` + 启动 `ACK_TIMEOUT` 定时器 | 收到 ACK → CHECK_COMPLETE；超时 → RETRY_ASSIGN |
| `RETRY_ASSIGN` | 重试分配 | 重发 `0x66`（最多 3 次） | ACK 收到 → CHECK_COMPLETE；超 3 次 → CHECK_COMPLETE |
| `CHECK_COMPLETE` | 检查是否还有更多从站 | 重置空闲超时计数器 | 连续 N 次无响应 → FINISHING；否则 → QUERYING |
| `FINISHING` | 收尾 | 发送 `0x68` + 保存从站列表到 Flash | 完成 → IDLE |

## 5. 从站信息表设计

### 5.1 内存结构

```c
/* 单个从站信息 */
typedef struct {
    uint8_t  station_addr;      // 站号 (1~255)
    uint8_t  is_online;         // 在线标志 (0x01=在线, 0x00=离线)
    uint16_t fault_count;       // 通信失败计数
    uint32_t last_seen;         // 最后通信时间戳 (sysTick)
    // 可扩展字段...
} SlaveInfo_t;

/* 从站信息表 */
#define MAX_SLAVE_COUNT     32      // 建议最大值（可根据实际调整）

typedef struct {
    uint8_t      slave_count;        // 当前从站总数
    uint8_t      reserved;           // 对齐保留
    uint16_t     crc;                // 整个表的 CRC16 校验
    SlaveInfo_t  slaves[MAX_SLAVE_COUNT];
} SlaveTable_t;
```

### 5.2 Flash 存储布局

利用现有的 `parameter.c/h` 参数存储机制，或在 Flash 末尾扇区开辟新的存储区域：

```
Flash 地址范围 (假设):
  0x0807F000 ~ 0x0807FFFF  (最后一个 4KB 扇区)

存储布局:
  ┌───────────────────────────────┐ 0x0807F000
  │  SlaveTable 版本号 (2 Bytes)  │
  ├───────────────────────────────┤
  │  从站总数 (1 Byte)            │
  ├───────────────────────────────┤
  │  保留 (1 Byte)                │
  ├───────────────────────────────┤
  │  从站信息表 (可变大小)         │
  │  SlaveInfo_t[slave_count]     │
  ├───────────────────────────────┤
  │  整个表的 CRC16 (2 Bytes)     │

```

## 6. 各阶段处理逻辑

### 6.1 触发寻址

```c
/**
 * @brief  启动自动寻址
 * @note   可由用户命令、按键、上电自动触发
 */
void AutoAddr_Start(void)
{
    if (g_GwAddrState != GW_ADDR_IDLE) {
        LOG_WARN("自动寻址已在运行中");
        return;
    }
    
    // 如果不需要保留已有映射，可以清除旧表重新分配
    // SlaveTable_Clear();
    
    g_GwAddrState = GW_ADDR_STARTING;
    g_NextStation  = 1;           // 从站号1开始分配
    g_QueryNoRespCount = 0;       // 连续无响应计数清零
    g_RetryCount = 0;             // 重试计数清零
    
    LOG_INFO("=== 开始自动寻址 ===");
    AutoAddr_SendStart();
}
```

### 6.2 发送 0x64 — 开始寻址

```c
void AutoAddr_SendStart(void)
{
    uint8_t frame[4];  // 地址 + 功能码 + CRC16(2)
    
    frame[0] = 0x00;                    // 广播地址
    frame[1] = FUNC_START_ADDR;         // 0x64
    uint16_t crc = ModbusCRC16(frame, 2);
    frame[2] = crc & 0xFF;
    frame[3] = (crc >> 8) & 0xFF;
    
    Set485Mode(1);
    UART_Send(frame, 4);
    Set485Mode(0);
    
    g_GwAddrState = GW_ADDR_QUERYING;
    osDelay(FRAME_GAP_MS);             // 帧间隔
    AutoAddr_SendQuery();               // 紧跟着发第一次查询
}
```

### 6.3 发送 0x65 — 查询下一个

```c
void AutoAddr_SendQuery(void)
{
    uint8_t frame[4];
    
    frame[0] = 0x00;
    frame[1] = FUNC_NEXT_QUERY;         // 0x65
    uint16_t crc = ModbusCRC16(frame, 2);
    frame[2] = crc & 0xFF;
    frame[3] = (crc >> 8) & 0xFF;
    
    Set485Mode(1);
    UART_Send(frame, 4);
    Set485Mode(0);
    
    // 启动超时定时器
    StartTimeoutTimer(QUERY_TIMEOUT_MS);  // 2000ms
    
    g_GwAddrState = GW_ADDR_QUERYING;
}
```

### 6.4 收到令牌响应处理

```c
/**
 * @brief  处理从站返回的令牌
 * @param  token  从站发回的 16 位随机令牌
 * @note   从站回复帧: [0x00] [Token_H] [Token_L] [CRC16(2)]
 */
void AutoAddr_OnTokenReceived(uint16_t token)
{
    StopTimeoutTimer();
    
    g_CurrentToken    = token;
    g_RetryCount      = 0;
    g_QueryNoRespCount = 0;         // 收到了响应，清零连续无响应计数
    
    // 进入分配阶段
    AutoAddr_SendAssign(token, g_NextStation);
}
```

### 6.5 发送 0x66 — 分配地址

```c
void AutoAddr_SendAssign(uint16_t token, uint8_t addr)
{
    uint8_t frame[7];
    
    frame[0] = 0x00;                    // 广播地址
    frame[1] = FUNC_ASSIGN_ADDR;        // 0x66
    frame[2] = (token >> 8) & 0xFF;     // Token 高字节
    frame[3] = token & 0xFF;            // Token 低字节
    frame[4] = addr;                    // 分配的站号
    
    uint16_t crc = ModbusCRC16(frame, 5);
    frame[5] = crc & 0xFF;
    frame[6] = (crc >> 8) & 0xFF;
    
    Set485Mode(1);
    UART_Send(frame, 7);
    Set485Mode(0);
    
    // 启动 ACK 超时定时器
    StartTimeoutTimer(ACK_TIMEOUT_MS);   // 1000ms
    
    g_GwAddrState = GW_ADDR_ASSIGNING;
    
    LOG_INFO("分配地址: Token=0x%04X, Addr=%d", token, addr);
}
```

### 6.6 收到 0x67 确认处理

```c
/**
 * @brief  处理从站返回的地址确认
 * @param  ackAddr  从站回复的站号
 * @note   回复帧: [分配的站号] [0x67] [站号] [CRC16(2)]
 */
void AutoAddr_OnAckReceived(uint8_t ackAddr)
{
    StopTimeoutTimer();
    
    // 记录到从站表
    if (SlaveTable_Add(ackAddr) == 0) {
        LOG_INFO("从站%d 确认成功!", ackAddr);
    } else {
        LOG_ERROR("从站表已满，无法记录站号%d", ackAddr);
    }
    
    g_NextStation++;                     // 下一个站号
    
    // 等待帧间隔后查询下一个
    osDelay(FRAME_GAP_MS);
    AutoAddr_SendQuery();
}
```

### 6.7 超时与重试处理

```c
/**
 * @brief  全局超时定时器回调
 * @note   在系统 Tick 或定时器中断中调用
 */
void AutoAddr_OnTimeout(void)
{
    switch (g_GwAddrState) {
        case GW_ADDR_QUERYING:
            // 查询超时：没有从站回复
            g_QueryNoRespCount++;
            LOG_INFO("查询超时(%d/%d)", g_QueryNoRespCount, MAX_IDLE_THRESHOLD);
            
            if (g_QueryNoRespCount >= MAX_IDLE_THRESHOLD) {
                // 连续 N 次无响应 → 认为所有从站已分配
                AutoAddr_Finish();
            } else {
                osDelay(FRAME_GAP_MS);
                AutoAddr_SendQuery();     // 再试一次
            }
            break;
            
        case GW_ADDR_ASSIGNING:
        case GW_ADDR_RETRY_ASSIGN:
            // ACK 超时：从站可能没收到分配帧或确认帧丢失
            g_RetryCount++;
            if (g_RetryCount <= MAX_RETRY) {
                LOG_WARN("ACK超时，重试(%d/%d)", g_RetryCount, MAX_RETRY);
                g_GwAddrState = GW_ADDR_RETRY_ASSIGN;
                AutoAddr_SendAssign(g_CurrentToken, g_NextStation);
            } else {
                // 重试耗尽，跳过这个从站
                LOG_WARN("从站分配失败(重试耗尽)，跳过");
                g_NextStation++;
                osDelay(FRAME_GAP_MS);
                AutoAddr_SendQuery();
            }
            break;
            
        default:
            break;
    }
}
```

### 6.8 发送 0x68 — 结束寻址

```c
void AutoAddr_Finish(void)
{
    uint8_t frame[4];
    
    frame[0] = 0x00;
    frame[1] = FUNC_END_ADDR;           // 0x68
    uint16_t crc = ModbusCRC16(frame, 2);
    frame[2] = crc & 0xFF;
    frame[3] = (crc >> 8) & 0xFF;
    
    Set485Mode(1);
    UART_Send(frame, 4);
    Set485Mode(0);
    
    // 保存从站表到 Flash
    SlaveTable_SaveToFlash();
    
    g_GwAddrState = GW_ADDR_IDLE;
    
    LOG_INFO("=== 自动寻址完成! 从站总数: %d ===", g_SlaveTable.slave_count);
}
```

```c
// 在 usart.c 的 485 帧接收完成处理中增加以下分发逻辑：

void Process485Frame(uint8_t *buf, uint16_t len)
{
    // CRC 校验（略）
    
    uint8_t addr = buf[0];
    uint8_t func = buf[1];
    
    // ===== 自动寻址帧分发 =====
    if (addr == 0x00) {
        // 广播地址的回复 — 可能是令牌响应 或 旧固件从站的其他广播回复
        
        if (func == FUNC_ADDR_ACK && len >= 4) {
            // 0x67 地址确认帧: [站号] [0x67] [站号] [CRC16]
            uint8_t ackAddr = buf[2];
            AutoAddr_OnAckReceived(ackAddr);
            return;
        }
        
        if (len == 4 && g_GwAddrState == GW_ADDR_QUERYING) {
            // 令牌回复帧: [0x00] [Token_H] [Token_L] [CRC16]
            uint16_t token = (buf[1] << 8) | buf[2];
            AutoAddr_OnTokenReceived(token);
            return;
        }
    }
    
    // ===== 非寻址帧，走原有协议处理流程 =====
    // ... 原有的 Modbus 处理逻辑 ...
}
```

## 9. 测试要点

### 9.1 单元测试场景

| 编号 | 场景 | 前置条件 | 预期结果 |
|------|------|---------|---------|
| T01 | 1个未配置从站 | 485 总线只接 1 个从站 | 成功分配站号 1，收到 ACK |
| T02 | 3个未配置从站 | 同时接入 3 个 | 分配 1、2、3，无碰撞死锁 |
| T03 | 无从站接入 | 485 空载 | 查询超时 N 次后结束，从站表为空 |
| T04 | 混合场景 | 2个已配置 + 2个未配置 | 已配置从站不响应，未配置的被分配 1、2 |
| T05 | 分配时 ACK 丢失 | 模拟从站收到 0x66 但网关收不到 0x67 | 网关重试 3 次后跳过，最终寻址完成 |
| T06 | 多次触发寻址 | 先完成一次寻址后再次触发 | 已有从站不重复分配或重新分配（取决于设计） |
| T07 | 从站表存满 | 已有 32 个从站 | SlaveTable_Add() 返回 -1，日志提示表满 |
| T08 | 网关断电重启 | 已保存从站表 | 上电加载从站表，站号不丢失 |

### 9.2 边界情况处理

| 场景 | 处理方式 |
|------|---------|
| 485 总线故障（短路/断路） | 所有查询超时，寻址自动终止，记录错误日志 |
| 令牌碰撞（CRC 错误） | 网关收到 CRC 错误的帧则丢弃，超时后重发查询 |
| 从站分配中途掉电 | 网关重试 3 次后跳过该从站，下次寻址时该从站仍为未配置状态 |
| 从站数量超过 MAX_SLAVE_COUNT | 分配完 MAX 个后，发送 0x68 结束，多余从站保持未配置 |
| 寻址过程中收到其他命令 | 建议寻址期间锁定 485 总线，不处理其他命令 |
| 已有从站和新从站混合 | 已配置从站站号保留，新从站在已有最大站号基础上继续分配 |

### 9.3 测试工具

- 可以用一个 **USB-485 转换器** + PC 串口调试工具模拟从站行为
- 也可以用多个开发板作为测试从站
- 推荐使用 **Modbus Poll / Modbus Slave** 工具辅助调试

### 10.4 CRC16 参考实现

```c
/**
 * @brief  计算 Modbus RTU CRC16
 * @param  buf  数据指针
 * @param  len  数据长度
 * @return CRC16 值（低字节在前）
 */
uint16_t ModbusCRC16(uint8_t *buf, uint16_t len)
{
    uint16_t crc = 0xFFFF;
    
    for (uint16_t i = 0; i < len; i++) {
        crc ^= buf[i];
        for (uint8_t j = 0; j < 8; j++) {
            if (crc & 0x0001) {
                crc = (crc >> 1) ^ 0xA001;
            } else {
                crc >>= 1;
            }
        }
    }
    return crc;
}
```
