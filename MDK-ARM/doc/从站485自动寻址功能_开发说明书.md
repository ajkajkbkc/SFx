## 1. 概述

### 1.1 功能目标

实现从站设备通过 RS-485 总线接入网关后，由网关**自动分配站号地址（1~N）**，无需人工干预拨码开关或手动配置。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **MCU 无关** | 不依赖特定 MCU 型号或硬件特性（如 Unique ID） |
| **最小改动** | 尽量复用已有协议栈，仅增加寻址相关处理 |
| **高兼容性** | 不影响原有 Modbus 通信功能 |
| **断电记忆** | 分配到的站号写入非易失存储，重启后保留 |

### 1.3 工作流程总览

```
上电 ─→ 检查是否已配置站号？
         ├── 已配置 → 加载站号，进入正常运行模式
         └── 未配置 → 站号设为 0x00，监听广播寻址命令
                       等待网关发起自动寻址流程
```

## 2. 硬件要求

### 2.1 最小硬件清单

| 资源 | 用途 | 说明 |
|------|------|------|
| UART 接口 | 485 通信 | 半双工模式，需控制 RE/DE 引脚 |
| 定时器 ×1 | 随机退避延时 | 精度要求不高，ms 级即可 |
| 非易失存储 | 保存站号 | 如 EEPROM、内部 Flash、FRAM 等 |
| GPIO ×1 | 485 方向控制 | 控制 MAX3485/SP3485 的 RE/DE 引脚 |

### 2.2 485 硬件连接参考

```
MCU                    MAX3485/SP3485
┌─────────┐           ┌──────────────┐
│ UART_TX ────────────→ DI            │
│ UART_RX ←─────────── RO             │
│ GPIO_CTRL ──────────→ RE (并联DE)   │
│          │           │              │
│          │           │   A ──── 485A
│          │           │   B ──── 485B
└─────────┘           └──────────────┘

GPIO_CTRL 控制逻辑:
  = 0: 接收模式 (RE=0, DE=0)
  = 1: 发送模式 (RE=1, DE=1)
```

## 3. 通信协议定义

### 3.1 帧格式

采用标准 **Modbus-RTU** 帧结构：

```
┌─────────┬─────────┬──────────────┬──────────┐
│ 地址域   │ 功能码  │   数据域      │  CRC16   │
│  1 Byte  │ 1 Byte  │   N Bytes    │  2 Bytes │
└─────────┴─────────┴──────────────┴──────────┘
```

- **地址域** = `0x00`（广播地址，所有从站必须接收处理）
- **CRC16** = Modbus 标准 CRC16（多项式 `0x8005`）

### 3.2 自定义功能码定义

| 功能码 | 宏定义名称 | 方向 | 数据域格式 | 描述 |
|--------|-----------|------|-----------|------|
| `0x64` | `FUNC_START_ADDR` | 网关→广播 | 无数据 | **开始自动寻址**：通知所有未配置从站进入寻址模式 |
| `0x65` | `FUNC_NEXT_QUERY` | 网关→广播 | 无数据 | **查询下一个**：轮询未配置从站，等待令牌响应 |
| `0x66` | `FUNC_ASSIGN_ADDR` | 网关→广播 | `[Token_H(1B)] [Token_L(1B)] [站号(1B)]` | **分配地址**：携带令牌和分配的站号 |
| `0x67` | `FUNC_ADDR_ACK` | 从站→网关 | `[站号(1B)]` | **地址确认**：从站确认已保存分配的站号 |
| `0x68` | `FUNC_END_ADDR` | 网关→广播 | 无数据 | **结束寻址**：通知所有从站退出寻址模式 |

> **注意**：从站回复 `0x67` 确认帧时，**地址域使用刚分配的站号**，而非广播地址 `0x00`。  
> 网关应当在发送 `0x66` 分配地址后，**切换地址过滤规则**，等待特定站号的回复。

### 3.3 时序参数

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 波特率 | 9600 / 19200 bps | 485 总线常用波特率 |
| 数据格式 | 8N1 | 8 数据位，无校验，1 停止位 |
| 随机延时窗口 | 10 ~ 2000 ms | 从站随机退避范围 |
| 响应超时 | 2000 ms | 网关等待从站最长响应时间 |
| 帧间隔 | ≥ 3.5 字符时间 | Modbus RTU 标准帧间隔 |

## 4. 从站状态机设计

### 4.1 状态定义

```
                                ┌──────────────────────────────────────────────┐
                                │                                              │
                                ▼                                              │
┌───────────┐  收到 0x64   ┌───────────┐  收到 0x65   ┌──────────────┐       │
│  ADDR_IDLE │────────────→│MODE_ACTIVE│────────────→│ RAND_DELAY   │       │
│  (空闲)    │              │(寻址模式)  │              │ (随机退避)    │       │
└───────────┘              └───────────┘              └──────┬───────┘       │
      ▲                                                      │               │
      │                                              ┌───────┴───────┐       │
      │                                              │               │       │
      │                                        总线空闲         监测到总线忙  │
      │                                              │               │       │
      │                                              ▼               │       │
      │                                        ┌───────────┐        │       │
      │                                        │WAIT_ASSIGN│────────┘       │
      │                                        │(等待分配)  │                │
      │                                        └─────┬─────┘                │
      │                                              │                      │
      │                                       收到 0x66                     │
      │                                     (令牌匹配)                      │
      │                                              │                      │
      │                                              ▼                      │
      │                                        ┌───────────┐                │
      │                                  ┌────→│ ADDR_DONE │──收到 0x68────┘
      │                                  │     │(已分配)   │
      │                                  │     └───────────┘
      │                                  │           │
      │                                  │     重新上电
      │                                  │           │
      │                                  │           ▼
      │                                  │     ┌───────────┐
      └────────── 收到 0x68 ─────────────┘     │ ADDR_IDLE │
                                               │(已配置站号)│
                                               └───────────┘
```

### 4.2 状态详细说明

| 状态 | 说明 | 进入条件 | 退出条件 |
|------|------|---------|---------|
| `ADDR_IDLE` | 空闲状态，正常通信模式 | 上电/复位/收到 0x68 | 站号未配置 + 收到 0x64 |
| `MODE_ACTIVE` | 已进入寻址模式，等待查询 | 收到 0x64 + 未配置 | 收到 0x65（→RAND_DELAY）或 收到 0x68（→IDLE） |
| `RAND_DELAY` | 随机退避延时中 | 收到 0x65 + 处于 MODE_ACTIVE | 延时到期且总线空闲（→WAIT_ASSIGN）或 监测到总线忙（→MODE_ACTIVE） |
| `WAIT_ASSIGN` | 已发送令牌，等待分配地址 | 随机延时到期 + 总线空闲 | 收到 0x66 且令牌匹配（→DONE）或 超时（→MODE_ACTIVE） |
| `ADDR_DONE` | 地址分配完成 | 收到 0x66 + 令牌匹配 + 保存成功 | 收到 0x68（→IDLE） |

## 5. 参数存储设计

### 5.1 存储结构

从站需要在非易失存储中保存以下信息：

| 偏移地址 | 大小 | 名称 | 说明 |
|---------|------|------|------|
| `0x00` | 1 Byte | `StationAddr` | 站号（`0x00`=未配置，`0x01~0xFF`=已配置） |
| `0x01` | 1 Byte | `ConfigFlag` | 配置标志（`0x5A`=已配置有效，其他值=未配置） |
| `0x02~0x0F` | 14 Bytes | 保留 | 预留扩展 |

### 5.2 读取/写入接口

```c
// 读取站号
uint8_t LoadStationAddr(void)
{
    if (ReadEEPROM(0x01) == 0x5A)      // 检查配置标志
        return ReadEEPROM(0x00);        // 返回已保存的站号
    else
        return 0x00;                    // 未配置，返回0
}

// 保存站号
void SaveStationAddr(uint8_t addr)
{
    WriteEEPROM(0x00, addr);            // 写入站号
    WriteEEPROM(0x01, 0x5A);           // 写入配置标志
}

// 清除配置（恢复出厂设置）
void ClearStationConfig(void)
{
    WriteEEPROM(0x00, 0x00);
    WriteEEPROM(0x01, 0x00);
}

// 判断是否已配置
uint8_t IsAddrConfigured(void)
{
    return (ReadEEPROM(0x01) == 0x5A) ? 1 : 0;
}
```

> **注意**：如果使用内部 Flash 存储，注意 Flash 的擦写寿命（通常 1 万~10 万次）。自动寻址是一次性操作，寿命足够。但若需频繁修改，建议使用外部 EEPROM。

## 6. 各功能码处理逻辑

### 6.1 0x64 — 开始自动寻址

```
接收条件: 地址域 == 0x00 (广播)
         功能码 == 0x64

处理流程:
  1. 检查 IsAddrConfigured() == 0?  // 仅未配置设备响应
     └── 否 → 忽略此帧
     └── 是 → 继续
  2. 将状态切换到 MODE_ACTIVE
  3. 记录日志: "进入自动寻址模式"
```

### 6.2 0x65 — 查询未配置设备

```
接收条件: 地址域 == 0x00 (广播)
         功能码 == 0x65
         当前状态 == MODE_ACTIVE (若非此状态则忽略)

处理流程:
  1. 生成随机延时 T = random(10 ~ 2000) ms
  2. 生成随机令牌 Token = random(0 ~ 65535)
  3. 保存 Token 到全局变量 g_TempToken
  4. 保存 T 到全局变量 g_RandDelay
  5. 启动定时器，定时值为 T
  6. 切换到状态 RAND_DELAY
```

### 6.3 退避延时到期处理（定时器回调）

```
触发条件: RAND_DELAY 状态下的定时器到期

处理流程:
  1. 检测 485 总线是否空闲？
     └── 总线忙（RX 线上有数据）→
         │  说明有其他从站先响应了
         │  切换到 MODE_ACTIVE
         │  等待下一轮 0x65 查询
         └── 返回
     └── 总线空闲 →
         │  发送 g_TempToken (2 字节，大端序)
         │  切换到 WAIT_ASSIGN
         │  启动超时定时器(1000ms)  // 防止丢帧死锁
         └── 返回

总线空闲检测方法:
  方法A: 检查 UART RX 接收完成标志是否在最近 5ms 内被置位
  方法B: 检查接收缓冲区是否有数据
  方法C: 使用硬件 CTS/空闲检测线（如有）
```

### 6.4 0x66 — 分配地址

```
接收条件: 地址域 == 0x00 (广播)
         功能码 == 0x66
         数据域长度 == 3 Bytes

数据域格式:
  Byte[0] = Token 高字节
  Byte[1] = Token 低字节
  Byte[2] = 分配的站号

处理流程:
  1. 从帧中提取 token_rx = (buf[2] << 8) | buf[3]
  2. 提取新站号 new_addr = buf[4]
  3. 对比 token_rx == g_TempToken ?
     └── 不相等 → 忽略此帧，返回
     └── 相等 → 继续
  4. 调用 SaveStationAddr(new_addr) 保存到非易失存储
  5. 构造回复帧:
     地址域 = new_addr（刚分配的站号）
     功能码 = 0x67
     数据域 = [new_addr]
     CRC16 = 计算(地址域 + 功能码 + 数据域)
  6. 切换到发送模式，发送回复帧（最多重试 3 次）
  7. 切换到状态 ADDR_DONE
  8. 记录日志: "寻址成功! 站号=%d", new_addr
```

### 6.5 0x68 — 结束自动寻址

```
接收条件: 地址域 == 0x00 (广播)
         功能码 == 0x68

处理流程:
  1. 无论当前处于什么状态（MODE_ACTIVE / RAND_DELAY / WAIT_ASSIGN / DONE）
     统统强制切换到 ADDR_IDLE
  2. 如果处于 WAIT_ASSIGN 状态（即尚未分配到地址）
     → 站号保持为 0x00（未配置），等待下一次自动寻址
  3. 记录日志: "退出自动寻址模式"
```

## 7. 关键函数接口定义

### 7.1 需要从站固件提供的底层接口

// =====================================================

/**

uint16_t GenerateRandom(void);

void StartOneShotTimer(uint16_t ms);

/**
 * @brief  停止当前运行的定时器
 */
void StopTimer(void);

/**
 * @brief  获取当前定时器剩余时间，0=已到期

uint16_t GetTimerRemaining(void);

 * @brief  切换 485 收发模式
 * @param  tx_mode  1=发送模式, 0=接收模式
 */
void Set485Mode(uint8_t tx_mode);

 * @brief  发送数据到 485 总线（含 CRC16）
 * @param  buf  待发送数据缓冲区

void Send485Frame(uint8_t *buf, uint16_t len);

/**
 * @brief  检测 485 总线是否空闲

uint8_t Is485BusIdle(void);

### 7.2 从站需要实现的核心逻辑接口

```c
// =====================================================
// 自动寻址核心逻辑
// =====================================================

/**
 * @brief  从站自动寻址状态枚举
 */
typedef enum {
    ADDR_IDLE       = 0,    // 空闲
    MODE_ACTIVE     = 1,    // 寻址模式已激活
    RAND_DELAY      = 2,    // 随机退避延时中
    WAIT_ASSIGN     = 3,    // 等待地址分配
    ADDR_DONE       = 4,    // 地址分配完成
} AddrState_t;

/**
 * @brief  全局状态变量（由各函数维护）
 */
AddrState_t  g_AddrState;      // 当前寻址状态
uint16_t     g_TempToken;      // 当前轮次的随机令牌
uint16_t     g_RandDelay;      // 当前轮次的随机延时(ms)
uint8_t      g_CurrentAddr;    // 当前站号（加载自存储）

/**
 * @brief  初始化自动寻址模块
 * @note   在设备上电时调用，加载已保存的站号
 */
void AutoAddr_Init(void)
{
    g_CurrentAddr = LoadStationAddr();
    g_AddrState = ADDR_IDLE;
    g_TempToken = 0;
    g_RandDelay = 0;
}

/**
 * @brief  处理接收到的 485 帧（主入口）
 * @param  buf   接收缓冲区（含地址域+功能码+数据+CRC）
 * @param  len   接收长度
 * @note   在 UART 接收完成回调中调用此函数
 */
void AutoAddr_ProcessFrame(uint8_t *buf, uint16_t len)
{
    uint8_t addr  = buf[0];       // 地址域
    uint8_t func  = buf[1];       // 功能码
    
    // 只处理广播帧 (0x00)
    if (addr != 0x00) return;
    
    // CRC 校验（由底层完成，假设传入时已验证）
    
    switch (func) {
        case 0x64:  Handle_StartAddr();   break;
        case 0x65:  Handle_NextQuery();   break;
        case 0x66:  Handle_AssignAddr(buf, len);  break;
        case 0x68:  Handle_EndAddr();     break;
        default:
            // 其他功能码按原有协议处理
            break;
    }
}

/**
 * @brief  定时器到期回调
 * @note   在 MCU 定时器中断或回调中调用
 */
void AutoAddr_OnTimerExpired(void)
{
    if (g_AddrState != RAND_DELAY) return;
    
    if (Is485BusIdle()) {
        // 总线空闲 → 发送令牌
        uint8_t frame[4];
        frame[0] = 0x00;                         // 广播地址回复
        frame[1] = (g_TempToken >> 8) & 0xFF;    // Token 高字节
        frame[2] = g_TempToken & 0xFF;           // Token 低字节
        // 注意：回复帧地址域为 0x00，网关通过令牌识别设备
        
        Set485Mode(1);                            // 切换到发送
        Send485Frame(frame, 3);                   // 发送
        Set485Mode(0);                            // 切回接收
        
        g_AddrState = WAIT_ASSIGN;
        StartOneShotTimer(1000);                  // 等待分配超时
    } else {
        // 总线忙 → 放弃本轮，等待下一轮查询
        g_AddrState = MODE_ACTIVE;
    }
}

/**
 * @brief  WAIT_ASSIGN 超时处理
 * @note   如果处于 WAIT_ASSIGN 状态且超时未收到分配指令
 *         （可能令牌帧被碰撞销毁），回退到 MODE_ACTIVE
 */
void AutoAddr_OnAssignTimeout(void)
{
    if (g_AddrState == WAIT_ASSIGN) {
        g_AddrState = MODE_ACTIVE;
    }
}
```

## 8. 集成与测试要点

### 8.1 代码集成步骤

| 步骤 | 内容 | 注意事项 |
|------|------|---------|
| 1 | 实现底层硬件接口 | ReadEEPROM / WriteEEPROM / GenerateRandom / 定时器 / 485方向控制 |
| 2 | 实现 CRC16 校验函数 | 必须与 Modbus 标准 CRC16 一致（多项式 `0x8005`） |
| 3 | 在 UART 接收完成回调中调用 `AutoAddr_ProcessFrame()` | 过滤广播地址 `0x00` |
| 4 | 在定时器中断中调用 `AutoAddr_OnTimerExpired()` | 注意中断优先级 |
| 5 | 在系统 Tick 或循环中处理 WAIT_ASSIGN 超时 | 防止死锁 |
| 6 | 上电调用 `AutoAddr_Init()` 加载已保存站号 | 在 Modbus 协议栈初始化前调用 |
| 7 | 完成后的通信使用分配的站号 | 替代固定站号 |

### 8.2 测试场景

| 测试编号 | 场景 | 预期结果 |
|---------|------|---------|
| T01 | 1个从站，未配置，上电后网关发起寻址 | 从站被分配站号1，保存成功 |
| T02 | 3个从站，全部未配置，同时接入 | 分配站号 1、2、3，无冲突 |
| T03 | 5个从站，已配置站号 2、5，其余未配置 | 在线的已配置从站不受影响，未配置的被分配新地址 |
| T04 | 从站分配地址后断电重启 | 上电后加载已保存站号，不再响应寻址 |
| T05 | 信号干扰导致令牌碰撞 | 碰撞后自动重试，最终全部分配成功 |
| T06 | 通信过程中突然断电再上电 | 已保存的站号不丢失 |
| T07 | 从站不支持寻址功能（旧固件） | 忽略所有 0x64~0x68 帧，正常工作不受影响 |

### 8.3 边界情况处理

| 情况 | 处理方式 |
|------|---------|
| 未配置从站始终收不到任何寻址帧 | 保持站号 0x00，不参与正常通信（可设超时后自动进入安全模式） |
| 随机延时中连续多轮碰撞 | 网关会持续发送 NEXT_QUERY，从站每次重新生成随机数，最终必然错开 |
| 保存站号时掉电 | 配置标志 `0x5A` 最后写入，写入失败则视为未配置，下次可重新分配 |
| 存储空间写入失败 | 返回 `0x67` 前检查保存结果，若失败则不回复 ACK，网关重新分配 |

### 9.3 随机数生成参考实现（LCG 算法）

```c
// 线性同余生成器，无需硬件 RNG
static uint32_t s_RandomSeed = 1;

void Random_Seed(uint32_t seed)
{
    s_RandomSeed = seed;
}

uint16_t GenerateRandom(void)
{
    // LCG 参数: LCG_MMC_64
    s_RandomSeed = s_RandomSeed * 6364136223846793005ULL + 1442695040888963407ULL;
    return (uint16_t)(s_RandomSeed >> 16);
}
```

> **提示**：可在首次上电时用 `SysTick` 或未初始化 RAM 的值做种子，确保每次复位后随机序列不同。
